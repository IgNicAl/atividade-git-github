# Atividade Git e GitHub

Este repositório foi criado para a atividade prática de Git e GitHub, com o objetivo de praticar comandos básicos de versionamento, criação de arquivos, commits, branches e envio de alterações para um repositório remoto no GitHub.

## Objetivo da atividade

A atividade consistiu em criar um repositório no GitHub, clonar o projeto para a máquina local, configurar o Git, criar arquivos de texto, realizar commits e trabalhar com branches.

Durante a execução, também foram praticados comandos no terminal Linux, uso do GitHub CLI, autenticação no GitHub e correção de erros comuns no processo de versionamento.

## Arquivos do repositório

O repositório contém os seguintes arquivos:

- `experiencias.txt`
- `planos-futuros.txt`
- `linguagens-favoritas.txt`

Além dos arquivos separados, os textos também estão registrados abaixo neste README.

## Minhas experiências acadêmicas e profissionais

Minha experiência acadêmica tem sido marcada pelo aprendizado prático na área de tecnologia. Durante esta atividade, trabalhei com conceitos importantes de Git e GitHub, como criação de repositório, clonagem de projeto, configuração de usuário, criação de arquivos, commits, branches e envio das alterações para um repositório remoto.

Além da parte teórica, tive contato direto com situações reais que podem acontecer em um ambiente de desenvolvimento. Ao usar o terminal no Fedora, precisei lidar com erros de configuração do Git, problemas de autenticação no GitHub, criação acidental de diretórios duplicados e ajustes no fluxo de trabalho antes de conseguir organizar corretamente o projeto.

Essa experiência mostrou a importância de entender não apenas os comandos, mas também o que cada erro significa. Mensagens como falta de identidade do Git, falha de autenticação por HTTPS e branches inexistentes ajudaram a compreender melhor o funcionamento do versionamento de código.

No aspecto profissional, considero que esse tipo de prática é muito importante, pois simula situações comuns em ambientes de desenvolvimento e DevOps. Saber usar Git, GitHub, terminal Linux e ferramentas de autenticação é essencial para trabalhar em equipe, organizar projetos e manter um histórico confiável das alterações realizadas.

## Minhas linguagens de programação favoritas

Minhas linguagens de programação favoritas são Python, JavaScript e Java. Escolhi essas linguagens porque elas são muito utilizadas no mercado e permitem criar diferentes tipos de projetos, desde scripts simples até sistemas completos.

Python é uma das minhas favoritas por ser uma linguagem simples, legível e muito versátil. Ela pode ser usada para automação, análise de dados, inteligência artificial, desenvolvimento web, scripts para Linux e várias outras finalidades. Para quem está aprendendo programação e tecnologia, Python ajuda bastante por ter uma sintaxe mais direta.

JavaScript também é uma linguagem que considero muito importante, principalmente por sua presença no desenvolvimento web. Com JavaScript é possível criar páginas interativas, aplicações front-end e também sistemas back-end usando Node.js. É uma linguagem essencial para quem deseja desenvolver aplicações modernas para a internet.

Java é outra linguagem que gosto por sua organização e por trabalhar fortemente com programação orientada a objetos. Ela é muito usada em sistemas corporativos, aplicações acadêmicas, APIs e também no desenvolvimento Android. Aprender Java ajuda a entender melhor estrutura de código, classes, objetos e boas práticas de desenvolvimento.

Essas linguagens são importantes para meus estudos porque cada uma contribui de uma forma diferente para minha formação. Python ajuda na praticidade, JavaScript aproxima do desenvolvimento web e Java fortalece a base em programação orientada a objetos e sistemas mais robustos.
